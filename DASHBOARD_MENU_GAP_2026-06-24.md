# Dashboard: gap-анализ главного и бокового меню (2026-06-24)

## Область сравнения
- Макет: `mwl_dashboard_desktop_dark.html`
- Реализация: `frontend/app/dashboard/page.tsx`, `frontend/components/Header.tsx`, `frontend/components/dashboard/Sidebar.tsx`
- Источник требований: `DESIGN_SYSTEM.md` (§10, §10a), `PLAN.md` (Фаза 1)

---

## 1) Зафиксированные требования (SSOT)

### 1.1 Главное меню / Header (authenticated shell)
1. Структура: `[Logo] [Site name] [Search field] [Lang] [Bell] [User]`.
2. Поиск должен быть отдельным полем в header (не в контенте страницы).
3. Header должен быть частью persistent shell и не должен пересоздаваться/скроллиться вместе с контентом.
4. RTL/HE должен зеркалить порядок корректно без «double-flip trap».

Основание:
- `DESIGN_SYSTEM.md` §10, блоки "Persistent shell" и "Header (authenticated)".
- `PLAN.md` Фаза 1: пункт `DashboardHeader — search field`.

### 1.2 Боковое меню / Sidebar
1. 3 группы пунктов:
   - Верх: Дашборд, Модули, Прогресс, Редактор
   - Центр: Формулы, Лаборатория, Достижения
   - Низ: Помощь, Настройки
2. Для пункта "Редактор" обязательно подменю:
   - Граф, Атом, Группы, Экзамены (в PLAN формулировка: Карта графа / Редактор атома / Группы / Примеры вопросников)
3. Логика подменю:
   - Открывается только при активном "Редактор"
   - Клик по Дашборд/Модули/Прогресс закрывает подменю и снимает активность "Редактор"
   - В collapsed-состоянии клик по "Редактор" должен раскрыть sidebar + открыть подменю
4. Состояние collapsed/expanded должно храниться в localStorage.
5. Visual states nav-item:
   - active: только цвет `#22D3EE` (иконка+текст), без фоновой плашки
   - hover: только цвет `#94a3b8`, без фоновой плашки
6. Иконки должны не «ездить» между collapsed/expanded.

Основание:
- `DESIGN_SYSTEM.md` §10, §10a (зафиксировано в июне 2026)
- `PLAN.md` Фаза 1 (`Sidebar ... Content submenu`, `localStorage`)

## 2) Что есть в макете

### Header
- Есть центрированный search input.
- Есть RU/HE, иконки уведомлений/профиля.
- Есть 3-колоночная структура header.

### Sidebar
- Есть 3 группы и пункт "Редактор".
- Есть подменю (`contentSubmenu`) с 4 подпунктами.
- Реализована логика `setActiveNav` + автодораскрытие sidebar при клике на "Редактор" в collapsed-состоянии.
- Подменю закрывается/открывается по активному состоянию.

## 3) Что есть в Next.js реализации сейчас

### Header
- Используется `Header mode="dashboard"`.
- Есть RU/HE, bell, avatar, theme toggle.
- Поискового поля в используемом header НЕТ.
- Компонент `DashboardHeader.tsx` с search существует, но в `dashboard/page.tsx` не подключен.

### Sidebar
- Есть 3 группы, но в верхней группе НЕТ пункта "Редактор".
- НЕТ подменю контент-конструктора.
- active/hover выполнены с фоновыми плашками (расходится с SSOT color-only).
- Collapse работает локально через `useState`, но НЕ сохраняется в localStorage.
- Иконки имеют другую геометрию/поведение, чем зафиксировано в макете.

## 4) Матрица расхождений (критично для Фазы 1)

1. Header search отсутствует в рабочем dashboard shell.
2. Sidebar не содержит "Редактор" и обязательное подменю.
3. Нет зафиксированной поведения подменю (open/close rules из §10a).
4. Нет persistence collapsed/expanded в localStorage.
5. Nav item states не соответствуют "color-only" правилу.
6. Непривязанный `DashboardHeader.tsx` создаёт дублирующую, но неиспользуемую реализацию.
